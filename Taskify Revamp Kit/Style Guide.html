Taskify Design System

A focused, restrained design language for fast software. One signal accent on a graphite canvas. Inter for prose, JetBrains Mono for code. Built around OKLCH color tokens and a 4px spacing grid, with dark mode at parity from the first byte.

Stack
  • CSS: custom properties + 8 component files (~50KB total)
  • JS: vanilla, attribute-driven (data-toggle, data-dropdown, …)
  • Templates: Laravel Blade components (<x-buttons.button/>)
  • Type: Inter (400/500/600/700/800), JetBrains Mono (400/500/600)
Foundations

Principles

Four rules that govern every component decision.

01

One signal, never two

A single brand accent—Signal Lime—earns every use. Status colors stay reserved for status. No decorative gradients.

02

Quiet by default

Borders & subtle surface shifts carry hierarchy. Shadows used sparingly; rounded corners stay tight (3–5px).

03

Monospace is meaningful

JetBrains Mono signals IDs, timestamps, counts, code—anything machine-readable. Reading code, not decoration.

04

Dark mode at parity

Every token has a dark counterpart. No "dark version" CSS overrides—just [data-theme="dark"] on <html>.

Forms

Multiselect (Ajax)

Tag-token multiselect with optional ajax-backed search. Type to filter, click to add, press Backspace on empty input to remove the last token. Supports both a local options array and a remote endpoint that returns JSON.

Local options

Try: design, frontend, backend. 
<x-forms.multiselect
    name="tags"
    placeholder="Pick tags…"
    :options="[
        ['value' => 'design',    'label' => 'design'],
        ['value' => 'frontend',  'label' => 'frontend'],
        ['value' => 'backend',   'label' => 'backend'],
        ['value' => 'auth',      'label' => 'auth'],
        ['value' => 'onboarding','label' => 'onboarding'],
    ]"
    :values="['design']"/>

Ajax search

Type to trigger a mock 250ms request. Real impl hits /api/users/search?q=…. 
<x-forms.multiselect
    name="user_ids"
    endpoint="/api/users/search"
    placeholder="Search teammates…"/>
{
  "results": [
    { "value": 12, "label": "Lin Tan" },
    { "value": 34, "label": "Priya Mehta" },
    { "value": 56, "label": "Devon Rios" }
  ]
}

Anatomy

The component is a styled wrapper around plain inputs — no JS framework dependency. Tokens are real <span class="tag-token"> elements that contain a hidden <input type="hidden" name="...">, so submission works with any backend that handles array form fields (Laravel: user_ids[]). 

<div class="multiselect" data-multiselect
     data-endpoint="/api/users/search"
     data-name="user_ids[]">
  <div class="tag-strip">
    <!-- tokens injected by JS -->
    <span class="tag-token" data-value="12">
      <span>Lin Tan</span>
      <button type="button" aria-label="Remove">×</button>
      <input type="hidden" name="user_ids[]" value="12"/>
    </span>
    <input type="text" placeholder="Search…"/>
  </div>
  <div class="dropdown-menu"><!-- async results --></div>
</div>

Props

PropTypeDefaultDescription
namestring—Hidden input name. Submitted as an array — append [].
endpointstringnullGET URL that returns { results: [{ value, label }] }. Receives ?q= query param.
optionsarraynullLocal options array. Use this OR endpoint, not both.
placeholderstring'Search…'Input placeholder.
valuesarray[]Pre-selected values (matched against options[].value).
