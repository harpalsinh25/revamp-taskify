Detailed Scope of Work: Platform Customization

This document outlines the detailed technical and functional specifications for the customization of the platform. The objective is to implement a robust, automated ecosystem that handles dynamic task management, complex role-based accessibility, automated KPI-driven payroll, and comprehensive daily reporting.

1. Task Management Customization

The core task module will be heavily customized to support dynamic data capture and streamlined daily reporting for assigned users.

Feature Specifications

  • Dynamic Task Forms & Templates: The creation interface will be overhauled. Instead of a one-size-fits-all form, users will select a "Task Template".
  • Contextual Field Rendering: Based on the selected template, the system's rendering engine will dynamically inject only the relevant custom fields required for that specific task type, hiding all non-essential default fields.
  • Daily Activity Submission (EOD): A new sub-module will be injected directly into the task view. This allows the assigned employee to log their End-of-Day (EOD) progress directly against the task without navigating away.
  • Advanced Reporting: Task reports will be upgraded to support deep filtering based on the dynamic custom fields, allowing management to query highly specific operational metrics.

2. User Registration & Accessibility Enhancements

The onboarding process and access control matrix will be modified to support specific organizational roles like 'Affiliate' and 'Affiliate Manager'.

Feature Specifications

  • Dynamic Registration Flow: The registration form will be enhanced to dynamically load required custom fields configured by the admin (e.g., Mobicash application details, regional data, or specific user preferences), allowing precise data capture right at signup without relying on static forms.
  • Specialized Role Configuration: The platform's foundational role architecture will be expanded.
  • Targeted Access Profiles: We will build, configure, and lock down specific permission profiles for Affiliates and Affiliate Managers, ensuring their dashboard views are strictly confined to their operational purview.

3. Automated KPI & Salary Integration

A completely automated, performance-driven payroll engine will be integrated, directly tying daily task performance to financial compensation.

Feature Specifications

  • Real-time Performance Dashboard: Every user profile will feature a new, dynamic KPI Card. This widget provides real-time visualization of their performance metrics (e.g., tasks completed vs. overdue, EOD logs submitted).
  • Automated Payroll Engine: The system will dynamically generate salaries and payslips by evaluating the user's base configuration against their actual KPI performance.
  • Performance-Based Adjustments (Bonuses/Penalties): A rule-based engine will automatically calculate financial modifiers. Users exceeding targets will receive calculated bonuses, while users missing deadlines or failing to submit EOD logs will incur automated salary penalties.

4. Daily Data Processing & Automation

To ensure all dashboards, reports, and KPI metrics remain perfectly accurate without manual intervention, a background automation engine will be deployed.

Feature Specifications

  • System-Wide Batch Processing: A cron-based scheduled engine will execute automatically at the close of every business day.
  • Automated Status Recalculations: The engine will scan the entire database, automatically flagging overdue tasks, recalculating total KPI scores, and refreshing the data caches that drive the frontend dashboards.
  • Automated Report Generation: The system will automatically compile the daily Affiliate Manager report—aggregating all transaction logs, application statuses, and overall performance metrics—and prepare it for management review the following morning.
