Plugin Architecture

A standardized structure for organized and maintainable plugins.

plugins/
                └── YourPlugin/
                    ├── 🎮 Controllers/           # Request handling
    │   └── PluginController.php
    ├── 🧠 Models/ # Data models
    │ └── PluginModel.php
    ├── 🗄️ Database/ # Migrations
    │ └── Migrations/
    ├── 🎨 Resources/ # Views & assets
    │ ├── views/
    │ └── lang/
    ├── 🚀 Providers/ # Service registration
    │ └── PluginServiceProvider.php
    ├── 🛣️ routes/ # Routes
    │ ├── api.php
    │ └── web.php
    ├── ⚙️ plugin.json # Metadata
    └── 📖 README.md # Documentation

Get Started in Minutes

Set up your plugin with a simple configuration process.

plugin.json JSON 
{
                            "name": "Your Plugin",
                            "slug": "your-plugin",
                            "description": "Extend application functionality",
    "version": "1.0.0",
    "enabled": true,
    "provider": "Plugins\\YourPlugin\\Providers\\PluginServiceProvider",
    "publish_tag": "your-plugin-assets"
    }

Configuration Fields

Field Description
name Plugin name
slug URL identifier
description Short description
version Semantic version
enabled Enable/disable flag
provider Service provider path
publish_tag Asset tag
